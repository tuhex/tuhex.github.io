.exe-Header Yapısı

Summary

.exe dosyalarının içerisindeki yapılara; gömülü kodların aslında ne olduğuna, ne işe yaradıklarına ve neden orada olduklarına bakacağız.

.exe Dosyaları

.exe uzantılı dosyalar Windows ve DOS sistemlerde 'executable file(çalıştırılabilir dosya)' olarak geçer. İçerisinde dosyanın .exe olmasını sağlayan, dosya ile ilgili özellikleri tutan, dosya verilerini tutan kısımlar bulunur. Windows sistemlerde bu yapıya PE/PE32, DOS sistemlerde MZ(ayrıca dos-header kodlarında imza harfleridir) denir. Bu yapılardaki fark oldukça fazladır. Fakat halihazırda MZ dosya kodlarının PE içerisine gömülü olduğunu söyleyelim.

Şimdi PE yapısını inceleyelim.

PE başlığı 64-bit uzunluğunda olan DOS-header ve DOS-stub dan oluşuyor. Her PE dosyası DOS-header ile başlar. Bu PE dosyasının kimliğidir. Konumuz DOS-header olduğu için alt kısımlara değinmeyeceğim.

DOS-header

DOS-header'ın içerisindeki kodlar WORD(2-byte) ve LONG(4-byte) veri tipinden oluşuyor. Bu özelliklerin her birisinin ismine Field diyebiliriz. Offset değerleri ise 0x00 ile başlıyor. 0x00 offset değerinin field ismi Signature. '5A' ve '4D' değeri ASCII'de 'M' ve 'Z' değerleridir. MZ, Mark Zbikowski isminden gelir.

0x00 offset değerinden başlayıp 0x40 offset değerine kadar gider. Arada ise DOS-header değerleri bulunur.

  • WORD e_magic // Magic Number (Signature)
  • WORD e_cblp // Bytes on last page of file
  • WORD e_cp // Pages in file
  • WORD e_crlc // Relocations
  • WORD e_cparhdr // Size of header in paragrahs
  • WORD e_minalloc // Minimum extra paragraphs needed
  • WORD e_maxalloc // Maximum extra paragraphs needed
  • WORD e_ss // Initial CS Value
  • WORD e_sp // Initial SP Value
  • WORD e_csum // Checksum
  • WORD e_ip // Initial IP value
  • WORD e_cs // Initial (relative) CS value
  • WORD e_lfarlc // File address of relocation table
  • WORD e_ovno // Overlay number
  • WORD e_res[4] // Reserved Words
  • WORD e_oemid // OEM Identifier
  • WORD e_oeminfo // OEM information
  • WORD e_res2[10] // Reserved words
  • LONG e_lfanew // File address for new exe header
